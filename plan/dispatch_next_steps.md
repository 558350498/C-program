# 调度系统下一阶段计划

## 目标

在现有 `TaxiSystem + ISpatialIndex + IDispatchStrategy` 骨架上，继续补齐离线数据回放和批量匹配能力，为后续接入 Kaggle CSV、tile 地图和最小费用最大流策略做准备。

## 下一阶段拆分

### 1. 数据标准化

- 定义请求、行程、车辆状态、tile 标识等内部统一类型
- 增加 CSV 预处理层，不让调度层直接依赖原始字段名
- 统一时间单位和坐标清洗规则

### 2. 空间离散化

- 先做固定 tile 网格
- 建立 `tile -> requests / free drivers` 状态容器
- 保留 KD-Tree 作为 tile 内或候选集合内的细筛能力

### 3. 事件驱动回放

- 做统一事件时间轴
- 支持请求到达、车辆完成订单、车辆重新空闲等事件
- 先用小时间窗口数据回放验证流程

### 4. 匹配策略扩展

- 保留当前最近车策略作为 baseline
- 新增批量匹配策略接口实现
- 先从局部候选集上的最小费用最大流开始，不做全图完全匹配

### 5. 指标与日志

- 记录匹配率、平均等待时间、平均接驾时间、未服务请求数
- 对批次构图和未服务决策补日志

## 备注

当前阶段的重点不是把算法复杂度推到极致，而是把：

- 数据格式
- 状态推进
- 候选边生成
- 目标函数

这四件事稳定下来。
